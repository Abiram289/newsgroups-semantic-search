Verdict — which method wins and why

Best overall
FCM + Cosine is the right choice for this system — not because it scores highest on every metric, but because it's the only method that's correct for the problem. Sentence embeddings encode meaning as direction, not magnitude. Cosine distance measures only direction; Euclidean conflates direction with document length. FCM's soft assignments mean a query about gun control legislation searches both politics and firearms clusters simultaneously, instead of being forced into one. KMeans would miss half the relevant documents.
FCM + Cosine
✓ Correct metric for embeddings. ✓ Multi-cluster search. FPC 0.1231 — finds real structure. 53 iterations to converge — genuinely difficult problem.
FCM + Euclidean
✗ Collapses to FPC = 0.0833 (minimum possible) in just 3 iterations. Every doc gets uniform memberships. No structure found — degenerate result.
KMeans + Cosine
Higher ARI (0.319) and NMI (0.506) — hard labels map cleanly to discrete categories. But hard assignment means single-cluster search only. Loses cross-topic queries.
KMeans + Euclidean
Similar scores to KMeans+Cosine (ARI 0.319, NMI 0.504) but assigns to different clusters than KMeans+Cosine — showing metric choice matters. Still hard assignment only.
